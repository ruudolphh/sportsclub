## 🔑 Key Concepts

### Version Pinning
```
❌ Before: django-cors-headers (ANY version)
✅ After:  django-cors-headers==4.9.0 (EXACT version)

Benefit: Reproducible builds. Same code = Same dependencies everywhere.
```

### Hash Verification
```
❌ Before: No cryptographic verification
✅ After:  --hash=sha256:3046a53b0e40d4b676c3b774c73411d7184ae2745fe8ce5e45c0f33d3ddb71a7

Benefit: Supply chain security. Detects tampered/malicious packages.
```

### Two-File System
```
requirements.in  ← What we want   (human-maintained)
           ↓
    pip-compile
           ↓
requirements.txt ← What we get    (auto-generated with hashes)
```

### Deterministic Builds
```
install 1: Django 6.0.2, Psycopg 3.3.2, Pydantic 2.12.5
install 2: Django 6.0.2, Psycopg 3.3.2, Pydantic 2.12.5 (IDENTICAL ✓)
install 3: Django 6.0.2, Psycopg 3.3.2, Pydantic 2.12.5 (IDENTICAL ✓)
```

## 🚀 Common Workflows

### Add New Dependency

```bash
1. echo "celery>=5.0" >> requirements.in
2. pip-compile requirements.in --generate-hashes
3. pip install -r requirements.txt --require-hashes --allow-unsafe
4. pytest                          # Test
5. git commit -am "Add celery"
```

See [QUICK_REFERENCE.md](QUICK_REFERENCE.md) for all commands.

### Respond to Vulnerability

```
1. Dependabot creates PR automatically
2. Reviews changes
3. CI tests run
4. Merge approved PR
5. Production deployment deploys secure version

See [dependencies.md](dependencies.md) section 10 for details.
```

### Update All Dependencies

```bash
pip-compile requirements.in --upgrade --generate-hashes
pytest -v
git commit -am "chore: Update dependencies"
```
